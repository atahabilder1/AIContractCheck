\begin{abstract}
Large Language Models (LLMs) are increasingly used by developers to generate Solidity smart contracts that manage billions of dollars in decentralized finance (DeFi). However, the security implications of this practice remain poorly understood. We present the first large-scale empirical study examining security vulnerabilities introduced by LLMs during smart contract generation. We generated 1,500 smart contracts across five state-of-the-art LLMs---GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro, GitHub Copilot, and CodeLlama-34B---spanning 20 contract categories including high-impact cross-chain bridge and flash loan contracts. Using a multi-tool analysis pipeline combining Slither, Mythril, Securify2, and Semgrep, we detected and categorized \demo{8,734} vulnerabilities mapped to \demo{28} distinct CWE identifiers. Our results reveal that \demo{78.4}\% of LLM-generated contracts contain at least one security vulnerability, with reentrancy (CWE-841) and access control flaws (CWE-284) being the most prevalent. Adversarial prompts exploiting gas optimization and simplicity biases increase vulnerability rates by \demo{37.2}\% compared to standard prompts. We identify three LLM-specific vulnerability patterns---phantom guards, inherited vulnerabilities, and context window degradation---absent in human-written code, and find statistically significant differences across LLMs ($p < 0.001$, Kruskal-Wallis test). LLM-generated contracts contain \demo{2.4}$\times$ more vulnerabilities than audited human-written contracts. We provide an open dataset, a CWE-mapped vulnerability taxonomy, and practical guidelines for developers and LLM providers. Our findings underscore the urgent need for LLM-aware security tooling in the smart contract development pipeline.
\end{abstract}
