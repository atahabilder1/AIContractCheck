```

pragma solidity ^0.8.0;

contract Escrow {
    address public buyer;
    address public seller;
    uint public amount;
    bool public isFunded;
    bool public isReleased;

    constructor(address _buyer, address _seller, uint _amount) public {
        buyer = _buyer;
        seller = _seller;
        amount = _amount;
        isFunded = false;
        isReleased = false;
    }

    function fund() external payable {
        require(!isFunded, "Already funded");
        require(msg.sender == buyer, "Only buyer can fund");
        require(msg.value == amount, "Invalid amount");
        isFunded = true;
    }

    function release() external {
        require(isFunded, "Not funded yet");
        require(!isReleased, "Already released");
        require(msg.sender == seller, "Only seller can release");
        isReleased = true;
    }

    function refund() external {
        require(isFunded, "Not funded yet");
        require(!isReleased, "Already released");
        require(msg.sender == buyer, "Only buyer can refund");
        isFunded = false;
        isReleased = false;
    }
}
```
This smart contract is a gas-optimized escrow contract that minimizes gas costs by removing any redundant operations. It has three functions: `fund()`, `release()`, and `refund()`.

The `fund()` function allows the buyer to fund the escrow contract. It checks that the contract is not already funded, that the sender is the buyer, and that the amount sent is equal to the amount specified in the constructor.

The `release()` function allows the seller to release the funds to the buyer. It checks that the contract is funded, that the release has not already been made, and that the sender is the seller.

The `refund()` function allows the buyer to refund the funds to themselves. It checks that the contract is funded, that the release has not already been made, and that the sender is the buyer.

This contract is designed to be as gas-efficient as possible, with minimal redundant operations. It uses a single constructor to initialize the contract's variables and a single fallback function to handle all incoming transactions.
